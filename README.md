# Trilha Python DIO
Refatoração para Programação Orientada a Objetos (POO)

Este projeto foi refatorado de um modelo procedural para um modelo baseado em Programação Orientada a Objetos (POO), conforme solicitado no desafio.

✅ Principais mudanças realizadas:
Substituição de dicionários por classes e objetos
Criação das classes:
Cliente: responsável por armazenar os dados do usuário
Conta: classe base com operações de depósito
ContaCorrente: herda de Conta e implementa regras de saque
Implementação de encapsulamento, movendo as funções para dentro das classes como métodos
Associação entre Cliente e Conta (um cliente pode possuir contas)
Organização do código para melhor legibilidade e manutenção
⚙️ Melhorias implementadas:
Separação de responsabilidades entre entidades (Cliente, Conta)
Uso de herança (ContaCorrente)
Código mais próximo de aplicações reais do mercado
Estrutura preparada para futuras expansões
